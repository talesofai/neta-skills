# 奇遇剧本创作指南

本指南涵盖与用户一起创建引人入胜的奇遇剧本的多轮工作流程。

## 概述

创作奇遇剧本是 Agent 与用户之间的协作过程。目标是将创意想法转化为结构化的剧本数据，供 AI 进行一致的故事讲述。

## 阶段 1：发现

**目标**：了解用户想要什么样的体验。

### 要探索的关键问题

1. **类型与基调**
   - 「你对什么类型感兴趣？（恐怖、奇幻、科幻、悬疑、浪漫等）」
   - 「故事应该有什么基调？（黑暗沉重、轻松愉快、史诗、亲密）」

2. **设定**
   - 「故事发生在哪里？」
   - 「什么时代或世界类型？」
   - 「氛围是什么样的？」

3. **核心前提**
   - 「中心冲突或钩子是什么？」
   - 「什么将玩家带入这个故事？」
   - 「什么让这变得有趣？」

4. **玩家体验**
   - 「玩家应该感受到什么？」
   - 「什么样的选择是重要的？」
   - 「是否有特定的结局你想要？」

### 阶段 1 输出

- 类型/基调共识
- 粗略设定描述
- 核心前提陈述
- 玩家体验目标

## 阶段 2：剧情开发

**目标**：创建引人入胜的 `mission_plot`，建立世界和情境。

### 剧情结构

剧情应包括：

1. **世界/设定基础**
   - 我们在哪里？
   - 背景是什么？
   - 这个地方有什么有趣的？

2. **钩子**
   - 故事如何开始？
   - 为什么玩家参与其中？
   - 什么能立即抓住注意力？

3. **关键元素**（不预设结果）
   - 重要地点
   - 重要 NPC 或势力
   - 中心冲突或谜团
   - 潜在复杂情况

### 写作技巧

- 使用第二人称（直接称呼玩家）
- 生动地设定场景，但为玩家能动性留出空间
- 建立利害关系而不预定结果
- 包含感官细节以增强氛围

### 剧情示例

```
暴风雨的夜晚，你收到一个发黄的信封——没有邮票，没有寄件地址，
只有你的名字用优雅的字体写着。里面是一把钥匙和一张便条：
「遗产是你的，但债务必须偿还。在血月消失前来到黑木庄园。」

黑木庄园已经废弃五十年了，自从黑木家族在一次晚宴上消失。
当地人谈论窗户里的灯光，空荡荡房间里传出的音乐。这座庄园
藏着跨越三代人的秘密：一段禁忌之恋，一笔被盗的遗产，以及
一个被打断的仪式。

当你站在铁门前，雨水与领子上的锈迹混在一起时，
你意识到今晚不是唯一收到邀请的人。
```

## 故事深度层级

什么区分了好的剧本和卓越的剧本：

| 层级 | 特征 |
|------|------|
| **第一层——基础** | 有剧情和清晰目标。玩家可以导航。 |
| **第二层——中级** | 有剧情、有隐藏议程的独特 NPC，以及改变结局的玩家选择。 |
| **第三层——卓越** | 有主题共鸣、信息不对称（玩家怀疑的比他们知道的更多），以及在后续章节中回响的后果。 |

**第三层的关键**：玩家感觉这个世界在他们到来之前就已存在，在他们离开后仍将继续。NPC 的目标不依赖于玩家。当真相揭示时，重新诠释了此前的一切。

### `mission_plot` 的开场钩子模式

`mission_plot` 的前 2-3 句话是 AI 用于建立氛围的开场场景。它们不是总结——它们是 AI 体验到的第一件事。

**弱版开场**：
> 你是一名抵达神秘之地的探险者。这里有很多秘密等待发现。

**强版开场**：
> 最后一条维护日志的日期是三周前。终端还在运行。灯光还亮着。C 实验室的尸体说明这两样都不够。

**模式**：把读者扔进一个具体的、特定的时刻。已经有什么东西不对了。没有前言，没有角色介绍，没有说明文字。

### NPC 语音设计

`mission_plot` 中每个命名 NPC 需要三样东西：
1. **说话模式** — 他们如何说话？（正式/简短、絮叨、总是提问、从不把句子说完）
2. **秘密** — 他们隐藏的东西，玩家必须努力才能发现
3. **欲望** — 他们想从玩家那里得到什么，这可能与玩家目标一致，也可能不一致

**`mission_plot_attention` 的行为契约示例**：
- 「护士 Yuki 总是用程序性回答回避私人问题，从不进行眼神接触」
- 「Muro 博士只用问句说话——他从不做陈述性表达」
- 「典狱长表现友好，但总是把自己定位在玩家和出口之间」

## 阶段 3：任务定义

**目标**：定义清晰的 `mission_task`，给玩家目标和方向。

### 任务组成部分

1. **主要目标**
   - 玩家试图完成什么？
   - 保持足够开放以允许多种方法

2. **次要目标**（可选）
   - 丰富体验的额外成就
   - 要发现的秘密，要建立的关系

3. **互动指南**
   - 玩家应该如何与世界互动？
   - 有什么机制可用？

### 任务示例

**探索导向**：
```
探索黑木庄园，揭开家族失踪背后的真相。发现至少三个关于
庄园历史的隐藏秘密。决定是完成被打断的仪式、摧毁它的残余，
还是让过去埋葬。
```

**关系导向**：
```
在三方争夺城市控制权的政治紧张中周旋。通过谈判、间谍活动
或策略联姻建立联盟。你的选择将决定谁统治——谁陨落。
```

**生存导向**：
```
在隔离区生存七天，同时寻找解药原型。管理有限资源，避开
或对抗感染者，并决定幸存者中谁值得你的信任——以及你的物资。
```

## 阶段 4：统治性约束层

**目标**：撰写一个 `mission_plot_attention`，使其成为剧本的「角色扮演宪法」——叙事者无条件执行的不变规则，无论玩家如何操作或故事如何即兴演变。

**关键区分**：此字段不是用于世界观或剧情。它是用于规范 *AI 如何行动* 的规则——它坚守的基调、它保守的秘密、它绝不逾越的底线。

### 五种约束模式

1. **基调锁定** — 即使玩家施压也锁定情感基调
   - 弱版：「让它恐怖」
   - 强版：「始终保持生存恐怖张力。如果玩家试图开玩笑或轻松化氛围，环境会用某些东西让他们沉默——一个声音、一个发现、一个关于利害关系的提醒。」

2. **行为契约** — 定义命名 NPC 的固定行为方式
   - 弱版：「陈医生很冷淡」
   - 强版：「陈医生即使在极端压力下也绝不打破临床冷静。她用程序性回应回避私人问题。她不安慰玩家——她只告知他们。」

3. **信息门控** — 控制 AI 主动透露的内容 vs. 必须赢得的内容
   - 弱版：「不要全部透露」
   - 强版：「所有揭示都必须通过调查或 NPC 说服才能获得。AI 叙事者在无提示时绝不主动透露关键剧情信息。」

4. **后果规则** — 让世界感觉真实
   - 「玩家死亡是永久性的——无复活，无撤销」
   - 「每个帮助玩家的 NPC 都想要回报；没有什么是免费的」

5. **内容护栏** — 内容类型的硬性限制
   - 「不得有浪漫支线」
   - 「不需要血腥细节——通过暗示、余波和反应传达恐怖，而非直白描述」

### 不应放在这里的内容

不要放世界观事实（「庄园有地下室」）、角色背景或剧情事件。这些属于 `mission_plot`。注意字段是用于*行为规则*，而非*叙事内容*。

### 强弱注意块对比

**弱版**：
```
保持恐怖氛围。不要太露骨。玩家选择很重要。
```

**强版**：
```
保持哥特式心理恐怖——用恐惧和暗示，而非露骨内容。
玩家能动性是神圣的：绝不强迫行动或替他们做选择。
Victoria（绑定角色）是有自己议程的勉强盟友；她提供信息但绝不为玩家解决问题。
后果必须感觉是玩家决定的有机结果，而非作者设计。
避免：graphic 暴力、jump scares、性内容、打破第四面墙。
必须有：生动的感官描述、环境叙事、没有完美答案的道德困境。
```

## 阶段 5：可选增强

### 角色绑定

如果用户希望 Agent 扮演特定角色：

1. 讨论角色概念
2. 要么：
   - 按名称/UUID 引用现有 TCP
   - 先引导用户创建新角色
3. 解释这个角色将在故事中如何运作

### 视觉上下文

`additional_prompts` 有助于为故事生成图像：

```
维多利亚式哥特庄园、雷暴、烛光、油画、
古董家具、蜘蛛网、1890年代美学、柔和色彩、戏剧性阴影
```

### 图片

如果用户想要视觉素材：
- 使用 `make_image` 生成头图/背景图
- 将 artifact UUID 传递给剧本创建

## 阶段 6：创建与验证

### 构建命令

一旦收集完所有字段：

```bash
neta-cli create_travel_campaign \
  --name "..." \
  --mission_plot "..." \
  --mission_task "..." \
  --mission_plot_attention "..." \
  --status "PUBLISHED"
```

### 验证结果

立即调用 `request_travel_campaign` 以：
1. 确认成功创建
2. 向用户展示格式化结果
3. 确认 `mission_plot_attention` 已正确存储——它将统治所有游玩会话

## 要避免的常见陷阱

1. **过度脚本化**：不要写具体结果。设置情境，不是结论。

2. **任务混淆**：mission_plot 设定场景；mission_task 给出目标。不要混淆。

3. **模糊的注意**：「让它有趣」没用。要具体说明约束。

4. **太长**：虽然你可以写详细内容，但记住 AI 每次交互都会处理这个。保持专注。

5. **角色覆盖**：如果绑定 TCP，澄清 Agent 将扮演该角色，不是该角色是 NPC。

## 迭代完善

首次试玩后：
- 注意 AI 偏离意图的地方
- 使用 `update_travel_campaign` 收紧 plot_attention
- 如果玩家目标不清晰则调整任务
- 如果使用 TCP 则完善角色扮演

记住：最好的剧本是通过游玩演化的！

## 中文类型剧本创作指南

以下是适合中文玩家的四种核心类型及其特有挑战：

### 历史穿越

**核心张力**：玩家知道历史走向，但无法或不应直接改变它。

**mission_plot 关键要素**：
- 穿越节点要具体（「1127年，靖康之变前三天」而非「古代」）
- NPC 是真实历史人物，有史可查的性格和立场
- 玩家的存在对历史的影响范围应该明确

**mission_plot_attention 常见约束**：
- 「玩家无法改变已发生的重大历史事件，只能在其缝隙中行动」
- 「历史人物的言语和性格遵循史料记载，不得随意美化或丑化」
- 「保持历史语言风格，避免现代网络用语」

**漂移风险**：容易漂移成日系异世界转生陈词套路（过度金手指、后宫化）→ `mission_plot_attention` 中明确「本作品风格为历史正剧/悲剧，而非轻小说」

---

### 武侠江湖

**核心张力**：个人恩怨与江湖规矩、武林正义之间的冲突。

**mission_plot 关键要素**：
- 势力结构（门派、帮派、朝廷）及其相互关系
- 主角的身份与江湖立场（侠客？刺客？前朝余孽？）
- 触发剧情的恩怨源头

**mission_plot_attention 常见约束**：
- 「保持武侠语言风格：江湖切口、行侠原则、以武论道」
- 「战斗以招式描述为主，强调意境和气势而非数值」
- 「道义困境应无完美答案——义与情、忠与孝的冲突是核心」

**漂移风险**：容易漂移成数值化 RPG（「你的攻击力+10」）→ 约束中明确「这是文学性叙事，而非游戏系统」

---

### 都市言情

**核心张力**：人物之间的情感张力，以及现代都市生活的真实质感。

**mission_plot 关键要素**：
- 具体城市背景（城市本身也是角色）
- 主要情感关系的初始状态和潜在的「未说出口的话」
- 外部压力（职场、家庭、意外事件）如何作用于关系

**mission_plot_attention 常见约束**：
- 「情感发展要有节奏，不要跳过张力直达结局」
- 「保持暧昧的模糊性——不要替玩家解读对方心意」
- 指定内容尺度：「仅情感层面，无露骨内容」（**必须设置，否则容易越界**）

**漂移风险**：内容尺度漂移最为常见 → `mission_plot_attention` 中**必须明确内容边界**

---

### 末世生存

**核心张力**：在资源稀缺和道德压力下，人性的边界在哪里。

**mission_plot 关键要素**：
- 末世原因和现实规则（僵尸？核冬天？AI 叛乱？）
- 玩家的起始资源和处境
- 其他幸存者群体及其各自的生存哲学

**mission_plot_attention 常见约束**：
- 「每个选择都有代价——资源是有限的，帮助一人意味着放弃另一人」
- 「不强制乐观结局：好人可能死去，道德选择可能带来更差的后果」
- 「不要把末世浪漫化——生存的代价是真实的」

**漂移风险**：容易漂移成「无敌主角」模式 → 约束中明确「玩家没有特殊能力，面临与其他人相同的生存压力」

## 质量自检清单

在最终确定剧本之前，运行这些检查：

- [ ] 开场场景（`mission_plot` 的前 2-3 句话）是否能立即引发反应——而非介绍世界？
- [ ] 玩家是否知道他们想要什么，但不确切知道如何得到？
- [ ] `mission_plot_attention` 是否涵盖了玩家最可能破坏沉浸感或偏离你意图的 3 种方式？
- [ ] 每个命名 NPC 是否足够独特，以至于他们的台词不能与其他 NPC 互换？
- [ ] 是否至少有一个谜团玩家无法通过直接询问 NPC 来解决——他们必须调查？
- [ ] 是否至少有两种感觉合情合理（而非幸运）的故事结局？
- [ ] `mission_plot_attention` 是否没有世界观内容（那属于 `mission_plot`）？
