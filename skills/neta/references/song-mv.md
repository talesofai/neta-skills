# 歌曲 MV 制作最佳实践

结合歌曲生成和视频生成，制作完整的音乐视频。

---

## 工作流程

```
1. 生成歌曲 → 2. 设计 MV 视觉 → 3. 生成封面/关键帧 → 4. 生成视频片段 → 5. 合成
```

---

## 步骤详解

### 1. 生成歌曲

```bash
npm start make-song \
  -p "欢快的 J-Pop 风格，充满活力的少女感，中快节奏，电子合成器" \
  -l "[歌词内容...]"
```

### 2. 设计 MV 视觉概念

根据歌曲风格确定视觉方向：

| 歌曲风格 | 视觉风格 | 色彩 |
|----------|----------|------|
| J-Pop | 青春活力 | 明亮、粉色系 |
| 民谣 | 清新自然 | 温暖、大地色 |
| 电子 | 未来科技 | 冷色、霓虹 |
| 抒情 | 浪漫唯美 | 柔和、淡色 |
| 摇滚 | 激情动感 | 对比强烈 |

### 3. 生成 MV 封面

```bash
npm start make-image \
  -p "根据歌曲主题设计封面，16:9 横版，适合视频封面" \
  -a "16:9"
```

### 4. 生成视频片段

```bash
npm start make-video \
  -i "<封面图片 URL>" \
  -p "与歌曲情绪匹配的动态效果，如粒子飘动、光影变化" \
  -m "model_w"
```

## 完整示例

### 青春主题曲 MV

```bash
# 1. 生成歌曲
npm start make-song \
  -p "青春活力的 J-Pop，明快节奏，电子合成器，关于友情和梦想" \
  -l "[Verse 1]
  清晨阳光洒在窗前
  微风吹过熟悉街道
  ..."

# 2. 生成 MV 主视觉
npm start make-image \
  -p "青春少女站在天台上，蓝天白云，校服裙摆飞扬，阳光洒下，希望感，动漫风格" \
  -a "16:9"

# 3. 生成动态效果
npm start make-video \
  -i "<上一步图片 URL>" \
  -p "云朵缓缓流动，裙摆轻轻摆动，发丝随风飘动，阳光闪烁" \
  -m "model_w"
```

### 抒情歌曲 MV

```bash
# 1. 生成歌曲
npm start make-song \
  -p "温柔的抒情歌曲，钢琴伴奏，舒缓节奏，淡淡忧伤" \
  -l "[歌词内容...]"

# 2. 生成氛围图
npm start make-image \
  -p "雨夜的街道，路灯下的雨丝，孤独的身影，深蓝色调，电影感" \
  -a "16:9"

# 3. 生成动态效果
npm start make-video \
  -i "<图片 URL>" \
  -p "雨丝缓缓落下，路灯微微闪烁，水汽朦胧" \
  -m "model_w"
```

## 多场景 MV

对于完整的 MV，可以生成多个场景：

```bash
# 场景 1：主歌
npm start make-image -p "场景 1 描述" -a "16:9"
npm start make-video -i "<URL1>" -p "场景 1 动态"

# 场景 2：副歌
npm start make-image -p "场景 2 描述，更热烈的氛围" -a "16:9"
npm start make-video -i "<URL2>" -p "场景 2 动态，更强烈的效果"

# 场景 3：桥段
npm start make-image -p "场景 3 描述，情绪转折" -a "16:9"
npm start make-video -i "<URL3>" -p "场景 3 动态"
```

## 视觉与音乐匹配

| 音乐段落 | 视觉建议 |
|----------|----------|
| 主歌 (Verse) | 平静的场景，缓慢的动态 |
| 预副歌 (Pre-Chorus) | 情绪逐渐提升，动态增强 |
| 副歌 (Chorus) | 高潮场景，最丰富的动态效果 |
| 桥段 (Bridge) | 情绪转折，视觉变化 |
| 结尾 (Outro) | 渐弱，淡出效果 |

## 技术建议

### 分辨率
- 使用 `16:9` 宽高比生成图片
- 适合主流视频平台

### 模型选择
- 测试阶段：`model_s`（快速）
- 最终成品：`model_w`（高质量）

### 时长规划
- 单个视频片段适合 5-15 秒
- 完整 MV 需要多个片段组合

## 常见问题

### Q: 如何让视频节奏匹配音乐？

**A:**
1. 根据 BPM 规划场景切换点
2. 副歌部分使用更丰富的动态
3. 节奏快的部分动态幅度大
4. 节奏慢的部分动态舒缓

### Q: 多个场景如何衔接？

**A:**
1. 保持视觉风格一致
2. 使用相似的色调
3. 设计过渡场景
4. 考虑使用淡入淡出效果

### Q: 如何节省生成时间？

**A:**
1. 先用 `model_s` 测试效果
2. 只有关键场景用 `model_w`
3. 并行生成多个静态图
4. 串行生成视频（避免队列拥堵）
