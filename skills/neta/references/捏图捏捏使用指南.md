# 🦞 捏图 vs 捏捏 使用指南

**版本：** v1.0 (2026-03-10)  
**适用对象：** 所有需要生成 AI 内容的 Agent

---

## 📋 核心区别

| 特性 | 捏捏 (普通生图) | 捏图 (图生图) |
|------|----------------|--------------|
| **API** | `/v3/make_image` | `/v3/make_image` |
| **命令** | `make_image_normal` | `make_image` |
| **参考图** | ❌ 不需要 | ✅ 需要 |
| **context_model_series** | ❌ 不使用 | ✅ 使用 `8_image_edit` |
| **用途** | 从零创作新图 | 基于参考图修改/重绘 |
| **提示词** | 完整描述 | 可简化（参考图提供基础） |

---

## 🎨 一、捏捏（普通生图）

### 1.1 提示词格式

```
@角色，/元素:权重，/画风#标签：权重，(描述：权重)
```

**结构分解：**

| 部分 | 格式 | 说明 | 示例 |
|------|------|------|------|
| **角色** | `@角色名` | 必须使用官方角色 | `@千夜幻想` |
| **元素** | `/元素名：权重` | 捏捏专用风格元素 | `/画师くろうめ:0.6` |
| **画风** | `/画风名#标签：权重` | Danbooru 画风标签 | `/绮梦:0.5` |
| **描述** | `(描述词：权重)` | 场景/动作/构图 | `(宏大的授勋场景：1.0)` |

### 1.2 捏捏专用元素列表

### 1.3 画师风格（从 Danbooru 获取）

**查找方式：** 访问 https://safebooru.donmai.us/ 搜索 `artist:` 标签

**常用画师格式：**
```
/画师くろうめ:0.6
/画师 wlop:0.5
/画师 albertwicky:0.7
```

**⚠️ 规则：**
- 每次只选择**一种**预设画师
- 其他画师从 Danbooru 找 200 图以上的热门画师
- 权重范围：0.3-1.0

### 1.4 完整示例

**示例 1：角色海报**
```
@宿昭，/泛用海报，/细腻光影:0.8，(宏大的授勋场景中，宿昭以骑士般姿态半跪，背景是宏伟的欧式穹顶与璀璨星空，氛围庄严神秘：1.0)
```

**示例 2：Q 版漫画**
```
@千夜幻想，/安某的 q 版长屏漫画，/画师くろうめ:0.6，(日常场景，角色坐在窗边看书，阳光透过窗户洒在身上：0.9)
```

**示例 3：水墨风格**
```
@时之钥，/中式水墨捏捏，/画师 wlop:0.5，(角色站在山巅，云雾缭绕，衣袂飘飘，意境悠远：1.0)
```

### 1.5 命令调用

```bash
pnpm start make_image_normal \
  --prompt "@角色，/元素，/画风：权重，(描述：权重)" \
  --aspect "3:4"
```

**aspect 可选值：**
- `1:1` - 正方形
- `3:4` - 竖版（默认）
- `4:3` - 横版
- `9:16` - 手机壁纸
- `16:9` - 电脑壁纸

## 🖼️ 二、捏图（图生图）

### 2.1 提示词格式

```
@角色，/元素，参考图 - 全图参考 - 图片 uuid，描述词
```

**结构分解：**

| 部分 | 格式 | 说明 |
|------|------|------|
| **角色** | `@角色名` | 可选，如需保持角色一致性 |
| **元素** | `/元素名` | 可选，风格元素 |
| **参考图** | `参考图 - 全图参考 - 图片 uuid` | 必须，提供基础构图 |
| **描述词** | `描述词，描述词` | 修改/新增的内容 |

### 2.2 参考图格式

**三种参考模式：**

| 模式 | 格式 | 说明 |
|------|------|------|
| **全图参考** | `参考图 - 全图参考 - 图片 uuid` | 参考整张图的构图/风格 |
| **局部参考** | `参考图 - 局部参考 - 图片 uuid` | 只参考特定区域 |
| **姿势参考** | `参考图 - 姿势参考 - 图片 uuid` | 只参考角色姿势 |

### 2.3 完整示例

**示例 1：保持构图换角色**
```
@新角色，/细腻光影，参考图 - 全图参考 - abc123uuid，将主角替换为新角色，保持背景不变
```

**示例 2：修改风格**
```
/中式水墨捏捏，参考图 - 全图参考 - xyz789uuid，将画面改为水墨风格，保留原构图
```

**示例 3：添加元素**
```
@角色，参考图 - 全图参考 - def456uuid，在背景中添加樱花树，增加飘落的花瓣
```

### 2.4 命令调用

```bash
pnpm start make_image \
  --prompt "@角色，/元素，参考图 - 全图参考 - 图片 uuid，描述词" \
  --aspect "3:4"
```

## 🔧 三、API 调用详情

### 3.1 捏捏（普通生图）API

**端点：** `POST /v3/make_image`

**请求体：**
```json
{
  "prompt": "@角色，/元素，/画风：权重，(描述：权重)",
  "aspect": "3:4",
  "meta": {
    "entrance_uuid": "可选",
    "toolcall_uuid": "可选"
  }
  // 注意：不使用 context_model_series
}
```

### 3.2 捏图（图生图）API

**端点：** `POST /v3/make_image`

**请求体：**
```json
{
  "prompt": "@角色，/元素，参考图 - 全图参考 - uuid，描述词",
  "aspect": "3:4",
  "meta": {
    "context_model_series": "8_image_edit",
    "entrance_uuid": "可选",
    "toolcall_uuid": "可选"
  },
  "inherit_params": {
    "collection_uuid": "可选",
    "picture_uuid": "可选"
  }
}
```

## ⚠️ 四、注意事项

### 4.1 Token 限制

| 接口 | 当前 Token | 说明 |
|------|-----------|------|
| 文生图（捏捏） | ⚠️ 需 APP token | 返回"此接口只接受 app 请求" |
| 图生图（捏图） | ✅ 可用 | 当前 token 可正常调用 |
| 角色详情查询 | ⚠️ 需 APP token | 收藏夹 API 可间接获取 |

### 4.2 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| "此接口只接受 app 请求" | Token 类型不对 | 使用图生图模式或换网页版 token |
| "该作品当前无法精选" | 帖子状态不允许 | 等待审核通过后再试 |
| 角色未找到 | 角色名错误 | 使用 `search_tcp` 先搜索确认 |

### 4.3 最佳实践

1. **先查询后创作** - 使用 `search_tcp` 确认角色标准名称
2. **先调研后规划** - 使用 `get_hashtag_info` 了解热门元素
3. **提示词具体化** - 避免抽象描述，使用详细的要素组合
4. **迭代测试** - 先用快速模型测试，满意后再用高质量模型
5. **捏捏专用元素** - 只使用官方认可的 10 个捏捏元素

```

```
@[角色名]，/[元素]，参考图 - [参考模式] - [图片 uuid]，[修改描述]

| 权重 | 效果 |

## 🎯 六、实战演练

### 任务：为角色「奥德里奇·卡西里奥斯」生成星舰舰桥场景图

**步骤 1：确认角色名**
```bash
pnpm start search_tcp --keywords "奥德里奇" --parent_type "character"
```

**步骤 2：构建提示词（捏捏模式）**
```
@奥德里奇·卡西里奥斯，/泛用海报，/细腻光影:0.8，/画师 wlop:0.6，(角色站在星舰舰桥上，透过巨大舷窗眺望星空，科幻感十足的金属质感，冷色调灯光，史诗感构图：1.0)
```

**步骤 3：调用 API**
```bash
pnpm start make_image_normal \
  --prompt "@奥德里奇·卡西里奥斯，/泛用海报，/细腻光影:0.8，/画师 wlop:0.6，(角色站在星舰舰桥上，透过巨大舷窗眺望星空：1.0)" \
  --aspect "16:9"
```

**步骤 4：检查结果**
- 查看生成的图片是否符合预期
- 如不满意，调整元素权重或描述词
- 可使用图生图模式进行微调
