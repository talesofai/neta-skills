# 视频生成最佳实践

适用于 `make-video` 命令。

---

## 提示词原则

### 1. 动作具体

描述明确的动作，而非抽象概念。

❌ **过于抽象：**
```
让她动起来
```

✅ **具体动作：**
```
角色轻轻挥手，头发随风飘动
```

### 2. 幅度适中

避免过于复杂的多重动作。

❌ **过于复杂：**
```
她先挥手，然后转身，跳起来，最后微笑
```

✅ **单一动作：**
```
角色温柔地挥手，脸上带着微笑
```

### 3. 时间暗示

可以暗示动作的速度。

✅ **带速度的描述：**
```
头发缓缓飘动
眼神慢慢转向一侧
轻轻眨眼
```

---

## 模型选择

| 模型 | 处理时间 | 适用场景 |
|------|----------|----------|
| `model_s` | 5-10 分钟 | 快速迭代测试、简单动作 |
| `model_w` | 10-20 分钟 | 最终成品、复杂动作/表情 |

**建议：** 先用 `model_s` 测试效果，满意后再用 `model_w` 生成高质量版本。

## 常见用例

### 角色动态立绘

```bash
npm start make-video \
  -i "https://example.com/character.jpg" \
  -p "角色轻轻呼吸，头发随风飘动，温暖的微笑，眼睛微微眨动" \
  -m "model_s"
```

### 动态壁纸

```bash
npm start make-video \
  -i "https://example.com/scenery.jpg" \
  -p "星空缓缓旋转，流星缓慢划过，梦幻的光点闪烁，宁静祥和" \
  -m "model_w"
```

### 视频封面动效

```bash
npm start make-video \
  -i "https://example.com/cover.jpg" \
  -p "标题文字微微发光，背景有粒子飘动，电影感" \
  -m "model_s"
```

### 从图片生成完整流程

```bash
# 1. 生成图片（16:9 适合视频）
npm start make-image \
  -p "梦幻的星空背景，流星划过，深蓝色调" \
  -a "16:9"

# 2. 从输出中获取图片 URL

# 3. 生成视频
npm start make-video \
  -i "<图片 URL>" \
  -p "星空缓缓旋转，流星缓慢划过" \
  -m "model_w"
```

## 提示词对比

### ❌ 动作过多
```
她挥手，转身，跳跃，然后微笑，头发飘动，衣服摆动
```

### ✅ 聚焦单一动作
```
她温柔地挥手，头发轻轻飘动，脸上带着微笑
```

### ❌ 抽象描述
```
让她变得生动
```

### ✅ 具体描述
```
她轻轻眨眼，嘴角微微上扬，发丝随风摆动
```

## 图片准备建议

### 适合的图片
- 清晰的主体轮廓
- 人物/物体占据主要画面
- 背景相对简洁
- 高分辨率（至少 512px 边长）

### 不适合的图片
- 多人物复杂场景
- 主体过小
- 过于复杂的背景
- 低分辨率模糊图片

## 常见问题

### Q: 视频动作不自然怎么办？

**A:**
1. 简化动作描述，一次只描述 1-2 个动作
2. 使用"轻轻"、"缓缓"、"慢慢"等词汇
3. 避免快速、剧烈的动作描述
4. 尝试使用 `model_w` 获得更好效果

### Q: 视频和原图差异太大？

**A:**
1. 提示词不要描述与原图冲突的内容
2. 避免描述大幅度的位置变化
3. 聚焦于细微的动态效果（呼吸、眨眼、飘动）

### Q: 生成时间太长？

**A:**
1. 测试阶段使用 `model_s`
2. 避免在高峰期批量提交
3. 单个任务完成后再提交下一个
