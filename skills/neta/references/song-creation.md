# 歌曲生成最佳实践

适用于 `make-song` 命令。

---

## 提示词结构

**推荐格式：**
```
[音乐风格] + [情绪/氛围] + [节奏/速度] + [乐器/元素] + [主题/场景]
```

**示例：**
```
欢快的 J-Pop 风格，充满活力的少女感，中快节奏，电子合成器和鼓点，关于友情和梦想的主题
```

## 歌词格式模板

```
[Verse 1]
主歌第一段，叙述故事背景
4-8 行，建议押韵

[Pre-Chorus]
预副歌，情绪铺垫
2-4 行

[Chorus]
副歌，高潮部分，记忆点
4-8 行，重复性强

[Verse 2]
主歌第二段，故事发展
4-8 行

[Chorus]
副歌重复

[Bridge]
桥段，情绪转折或升华
2-4 行

[Chorus]
副歌最后一次（可升调）

[Outro]
结尾，渐弱
2-4 行
```

## 常见用例

### 角色主题曲

```bash
npm start make-song \
  -p "角色专属主题曲，J-Pop 风格，活泼可爱，魔法少女感，电子合成器，明快节奏" \
  -l "[歌词内容...]"
```

### 背景音乐

```bash
npm start make-song \
  -p "轻柔的钢琴背景音乐，宁静安详，适合阅读，舒缓节奏" \
  -l "[纯音乐可省略歌词或使用简单的哼唱]"
```

### 战斗曲

```bash
npm start make-song \
  -p "激昂的战斗曲，摇滚风格，电吉他，强烈鼓点，热血沸腾" \
  -l "[Verse 1]
  燃烧吧斗志
  冲破黑暗黎明
  ..."
```

### 抒情歌曲

```bash
npm start make-song \
  -p "温柔的抒情歌曲，民谣风格，吉他伴奏，清新自然，淡淡忧伤" \
  -l "[Verse 1]
  雨后的街道
  一个人走着
  ..."
```

## 歌词写作技巧

### 1. 押韵

句尾押韵让歌曲更流畅：
```
清晨的阳光洒在窗前 (qián)
微风吹过熟悉的街道 (dào)
想起昨天你的笑脸 (liǎn)
心中涌起温暖感觉 (jué)
```

### 2. 意象具体

使用具体的意象而非抽象概念：
- ❌ "我很开心"
- ✅ "阳光洒在窗前，微风吹过街道"

### 3. 情感递进

从主歌到副歌情感逐渐升华：
```
Verse: 叙述故事
Pre-Chorus: 情绪铺垫
Chorus: 情感爆发
```

### 4. 重复记忆点

副歌要有容易记住的核心句：
```
一起走过春夏秋冬
手牵手看遍世间风景
```

## 常见问题

### Q: 歌词太长怎么办？

**A:** 
- 精简到 3500 字符以内
- 优先保留主歌和副歌
- 可以省略部分重复段落

### Q: 纯音乐如何生成？

**A:**
- 歌词可以填写简单的哼唱词（如"啦~"）
- 或者填写简短的氛围描述
- 重点在 prompt 中描述音乐风格

### Q: 如何保持风格一致性？

**A:**
- 保存成功的 prompt 模板
- 使用相似的风格描述词汇
- 参考同类型的成功作品
