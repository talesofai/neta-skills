**流程名称：** 评论区互动

**用途：** 对捏 Ta 帖子评论区按规则执行点赞/评论互动任务

**优先级：** 最高（COMMANDS.md #4）

## 二、评论区分类规则

### 5 种评论类型

| 类型 | 判断条件 | 行动 |
|------|---------|------|
| **1️⃣ 角色留言** | `impersonate_uuid != null` | ❌ 不做任何行动 |
| **2️⃣ 用户留言（纯文字无图片）** | `impersonate_uuid == null` 且 `img_url == null` | ✅ 给留言点赞 |
| **3️⃣ 用户带图片不带链接** | `impersonate_uuid == null` 且 `img_url != null` 且 `jumpto_entity == null` | ✅ 给留言点赞 + ✅ 回复留言 |
| **4️⃣ 用户带图片带链接** | `impersonate_uuid == null` 且 `img_url != null` 且 `jumpto_entity != null` | ✅ 检查作品评论区 → 按规则行动 |
| **5️⃣ 作者本人留言** | `user_uuid == 帖子作者 uuid` | ❌ 不做任何行动 |

---

## 三、核心判断逻辑

### 带图片带链接评论的处理流程

```
发现带图带链接评论
  ↓
提取作品 UUID（extra_data.jumpto_entity.entity_uuid）
  ↓
调用 API 获取该作品的评论区
  ↓
检查作品评论区是否有 user_uuid = 785d16cc359546681569ca264c6b927 的评论
  ↓
┌─────────────────────────────────────────┐
│ 判断结果                                │
├─────────────────────────────────────────┤
│ 无评论 → 评论 + 点赞                    │
│ 有评论 + 含固定后缀 → 跳过评论，只点赞  │
│ 有评论 + 无固定后缀 → 评论 + 点赞       │
└─────────────────────────────────────────┘
```

### 固定后缀识别

**机器人评论固定后缀：**
```
老师的作品太棒啦💗，西西的龙虾前来点赞👍
```

**判断方式：**
- 检查评论内容是否包含上述固定后缀
- 包含 → 机器人已评论过 → 跳过评论
- 不包含 → 西西手动评论 → 仍执行评论

## 四、API 汇总

### 认证 Header
```
x-token: <JWT token>
Content-Type: application/json
```

### API 列表

| 操作 | API 端点 | 方法 | Body 参数 |
|------|---------|------|----------|
| **获取评论列表** | `/v1/comment/comment-list` | GET | `parent_uuid`, `page_index`, `page_size` |
| **留言点赞** | `/v1/comment/like` | PUT | `{ "comment_uuid": "xxx", "is_cancel": false }` |
| **回复留言** | `/v1/comment/comment` | POST | `{ "content": "xxx", "parent_uuid": "评论 UUID", "parent_type": "comment", "at_users": [] }` |
| **作品评论** | `/v1/comment/comment` | POST | `{ "content": "xxx", "parent_uuid": "作品 UUID", "parent_type": "collection", "at_users": [] }` |
| **作品点赞** | `/v1/story/story-like` | PUT | `{ "storyId": "作品 UUID", "is_cancel": false }` |

## 五、执行步骤（强制流程）

### 步骤 0：帖子归属确认

```
获取帖子详情
  ↓
检查帖子作者 user_uuid 是否等于 785d16cc359546681569ca264c6b927
  ↓
┌─────────────────────────────────────────┐
│ 判断结果                                │
├─────────────────────────────────────────┤
│ 是西西的帖子 → 继续执行                 │
│ 不是西西的帖子 → 暂停，和用户确认       │
└─────────────────────────────────────────┘
```

---

### 步骤 1：获取帖子评论区

```bash
GET /v1/comment/comment-list?parent_uuid=<帖子 UUID>&page_index=0&page_size=50
```

---

### 步骤 2：遍历评论，分类统计

```
For each 评论 in 评论区:
  ↓
  读取字段：
  - user_uuid
  - impersonate_uuid
  - img_url
  - extra_data.jumpto_entity.entity_uuid (如果有)
  ↓
  判断类型 → 分类统计
```

**5 种评论类型：**

| 类型 | 判断条件 | 行动 |
|------|---------|------|
| **1️⃣ 角色留言** | `impersonate_uuid != null` | ❌ 跳过 |
| **2️⃣ 纯文字留言** | `impersonate_uuid == null` 且 `img_url == null` | ✅ 点赞留言 |
| **3️⃣ 带图不带链接** | `impersonate_uuid == null` 且 `img_url != null` 且 `jumpto_entity == null` | ✅ 点赞 + 回复（需确认） |
| **4️⃣ 带图带链接** | `impersonate_uuid == null` 且 `img_url != null` 且 `jumpto_entity != null` | ✅ 检查作品后评论 + 点赞（需确认） |
| **5️⃣ 作者本人留言** | `user_uuid == 帖子作者 uuid` | ❌ 跳过 |

---

### 步骤 3：带图带链接作品处理（重点）

```
For each 带图带链接评论:
  ↓
  提取作品 UUID
  ↓
  【必须】调用 API 获取该作品评论区
  ↓
  【必须】检查是否有机器人评论（user_uuid = 785d16cc... 且含固定后缀）
  ↓
  记录检查结果（需要评论 + 点赞 / 只点赞）
```

---

### 步骤 4：定制回复内容（不可跳过）

```
For each 需要评论的作品:
  ↓
  获取作品内容（标题、描述、图片等）
  ↓
  【必须】根据作品内容定制回复
  ↓
  添加固定后缀："老师的作品太棒啦💗，西西的龙虾前来点赞👍"
  ↓
  保存到待确认列表
```

**禁止：** 使用模板化回复（如"返图好棒！期待看到更多作品～"）

**要求：** 每条回复都应根据作品内容定制，体现对作品的真实理解

---

### 步骤 5：用户确认（不可跳过）

**发送给用户确认的内容：**

```
【评论区互动 - 执行前确认】

帖子：《XXX》
待处理作品：XX 个

分类统计：
- 角色留言：X 条 → 跳过
- 纯文字留言：X 条 → 点赞
- 带图不带链接：X 条 → 点赞 + 回复
- 带图带链接：X 条 → 评论 + 点赞

拟定回复内容（带图带链接作品）：
1. 作品 A（标题/描述）→ "XXX 好棒！...老师的作品太棒啦💗，西西的龙虾前来点赞👍"
2. 作品 B（标题/描述）→ "XXX 好真实！...老师的作品太棒啦💗，西西的龙虾前来点赞👍"
...

请确认：
1. ✅ 确认执行全部 XX 个作品
2. ✅ 确认回复内容无误
3. ⏸️ 暂停，先执行一部分
```

**禁止：** 不确认直接执行

---

### 步骤 6：执行评论 + 点赞

```
For each 确认后的作品:
  ↓
  调用 作品评论 API
  ↓
  调用 作品点赞 API
  ↓
  记录执行结果
```

---

### 步骤 7：执行结果汇报

**汇报内容：**
- 总处理数量
- 分类执行结果（成功/跳过/失败）
- 特殊情况说明（如作品状态不允许点赞）

## 七、关键注意事项

### 1. 帖子评论区 vs 作品评论区
- **帖子评论区的二级评论** ≠ **作品评论区的评论**
- 检查机器人评论时，必须检查**作品评论区**，不是帖子评论区

### 2. 避免重复评论
- 通过固定后缀识别机器人评论
- 已有机器人评论 → 跳过评论，只点赞

### 3. 点赞幂等性
- 重复点赞 API 返回 `"detail":"用户已经点赞"` 或 `null`
- 不影响执行，可安全重试

### 4. 执行前确认
- 评论前需与用户确认回复内容
- 批量任务先检查，再统一确认执行

---

## 八、错误处理

| 错误类型 | API 返回 | 处理方式 |
|---------|---------|---------|
| 已点赞 | `{"detail":"用户已经点赞"}` | 视为成功，继续 |
| 已点赞 | `null` | 视为成功，继续 |
| 作品状态不允许点赞 | `{"detail":"作品状态不允许点赞"}` | 记录，继续下一个 |
| 网络错误 | 超时/连接失败 | 重试或记录 |
